# P4 re-gate: 100% hallucination-free — 2026-07-04

Second measured pass at the P4 signal (base-vs-tuned hallucination-free rate
on the 111-task benchmark), after growing the corpus 335 → 1030 → 1309 and
fixing the three failure classes the first re-run exposed.

## Results (Qwen2.5-Coder-0.5B-Instruct, LoRA, greedy, A1 prompt)

| arm | valid JSON | hallucination-free |
|---|---|---|
| base | 100/111 (90%) | **0/111 (0%)** |
| tuned, 335-corpus (07-04 morning) | 100% | 32/111 (28%) |
| tuned, 1030-corpus | 111/111 (100%) | 99/111 (89%) |
| tuned, 1309-corpus | **111/111 (100%)** | **111/111 (100%)** |

Same scoring as `eval_gate.py` (valid JSON; every `Var` is a `p<N>` param or
an in-scope symbol), run batched (`eval_gate_batched.py`, BS=32, left-padded
greedy — a 3090 finishes in ~4 min instead of ~40).

## The path from 28% → 100%

**1030 corpus** (`corpus-full.jsonl`: literal-constant, conditional,
multi-def shapes added): train loss 0.90 → 0.10, gate 28% → 89%.

The 12 remaining failures fell into exactly three classes
(`eval_failures.py` dumps them):

1. **Literal-as-Var (8)** — `{"Var":"10"}` instead of `{"Lit":{"Int":10}}`.
   The corpus had op+literal shapes, but never the benchmark's phrasing
   ("(parameter p0 named x) … computes Nat.max of x and 10"), so "10" was
   treated as a name.
2. **Invented library symbol (2)** — `Nat.clamp` with only min/max in scope.
3. **Prompt-name leak + arity (2)** — "named a, b, c" pulled `a, b, c` into
   the body instead of the p-pool, with a flat 3-arg `App` on a binary op.

**+279 targeted examples** (`gen_targeted.py` → `targeted.jsonl`,
`corpus-v2.jsonl` = 1309): bench-style phrasings and shapes with
**disjoint names/constants** (floor_at_6 vs atleast5, bound/restrict vs
clamp, biggest3 vs max3 — no (name, op, K) overlap with any bench family, so
the gate still measures shape generalization, not memorization). Retrain
(4 epochs, ~3.5 min, loss → 0.085): all three classes gone, 111/111 clean.

## What this is / isn't

- **Is:** the strongest cold-start signal yet — a $0.05, 4-minute fine-tune
  takes a 0.5B model from 0% to 100% hallucination-free on the benchmark,
  and the last 11 points came from *shape coverage*, not memorized answers.
- **Isn't:** the full P4 gate (tuned-on-Claw Pass@1 ≥ stock-on-Python at
  repo level). That still needs functional grading of the tuned model
  (compile + contracts, not just clean references) and the Python-parity
  comparison. hallucination-free ≠ correct.
- The benchmark and corpus are both synthetic and share generator DNA;
  disjoint names keep it honest, but a held-out, human-written task set is
  the obvious next hardening step.
