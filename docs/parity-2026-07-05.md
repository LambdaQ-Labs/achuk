# P4 parity: tuned-Achuk beats stock Python, JS, Go and Rust — 2026-07-05

The master plan's win condition (the "Matthew-effect reversal"): a model
fine-tuned on Achuk's synthetic corpus must outperform the SAME model
writing incumbent languages it saw billions of tokens of. Measured today,
functional Pass@1, five arms, one checkpoint.

## Setup

- **Model:** Qwen2.5-Coder-0.5B-Instruct, both ways. The Achuk arm uses the
  corpus-v4 LoRA (1,661 synthetic examples, ~4 min / ~$0.03 of GPU); the
  Python/JavaScript/Go/Rust arms use the stock checkpoint.
- **Tasks:** all 116 benchmark tasks with executable contracts (synthetic
  + held-out human-written). Non-Achuk arms get a semantics glossary
  (`Nat.max = max`, saturating sub, …) — the same information the Achuk arm
  gets from its scope list.
- **Grading:** identical for every arm. The same test-case generator the
  Achuk grader uses (integer grid, precondition-filtered); the produced
  function is EXECUTED (CPython, node, `go run`, `rustc -O`; Achuk via
  `achuk defs-grade` → interpreter) and every contract is checked against
  real results. Equal charitable normalization for all arms: code-fence
  stripping, truncation repair (balanced-brace / AST-trim), Go package-decl
  merge. Wrong-but-complete code is never touched.

## Results (functional Pass@1, greedy)

| arm | pass@1 |
|---|---|
| **Achuk (tuned)** | **110/116 (94%)** |
| JavaScript (stock) | 104/116 (89%) |
| Python (stock) | 66/116 (56%) |
| Rust (stock) | 41/116 (35%) |
| Go (stock) | 9/116 (7%) |

**The gate passes against all four.** The same 0.5B that writes 56%-correct
Python and 7%-correct Go writes 94%-correct Achuk after a $0.03 fine-tune on
data no human wrote.

The v4 adapter also re-passes the reference gate: 121/121 valid,
hallucination-free, effect-sound on the synthetic benchmark.

## Reading it honestly

- **0.5B is a toy.** Stock arms are weak at everything (Go collapses on
  int64-typed micro-functions; Rust drowns in truncated doc-test
  boilerplate). The claim is NOT "Achuk beats Python for coding" — it is
  the cold-start thesis at toy scale: on a benchmark with verifiable
  semantics, purpose-built synthetic data beats pretraining mass. Whether
  the effect survives at 7B/32B against much stronger stock baselines is
  THE open question, and it now has a harness.
- **Tasks are micro-functions** (arithmetic, clamps, conditionals) with
  integer contracts. Real development — I/O, data structures, multi-file —
  is not yet represented; corpus v4 widened shapes, the benchmark must
  follow.
- Achuk's 6 failures are all held-out tasks (clamp01, guard-dec, inc3,
  larger, quadruple, succ-sqr) — the generalization edge, same tasks that
  are hardest in every language.
- Benchmark and corpus share generator DNA for the synthetic subset;
  names/phrasings are disjoint and the held-out set is included, but
  third-party tasks remain the next hardening step.

## The 7B run (same day, same harness)

The open question above — does the effect survive against a much stronger
stock baseline? — measured the same afternoon. Qwen2.5-Coder-7B-Instruct,
both ways: the Achuk arm fine-tuned on the same corpus-v4 (54 min, ~$0.20,
train loss 0.039), the other arms stock.

| arm (same 7B model) | pass@1 | (0.5B was) |
|---|---|---|
| **Achuk (tuned)** | **110/116 (94%)** | 94% |
| Rust (stock) | 102/116 (87%) | 35% |
| Go (stock) | 99/116 (85%) | 7% |
| Python (stock) | 83/116 (71%) | 56% |
| JavaScript (stock) | 80/116 (68%) | 89% |

**The gate passes at 7B against all four.** The margin collapsed as
predicted — stock Rust jumps 35→87, Go 7→85 — but Achuk holds 94% and stays
on top. Notably: (a) the fine-tune's value persists even when the base
model is strong; (b) Achuk's six failures are the same held-out tasks at
both scales (clamp01, guard-dec, inc3, …) — a corpus-coverage signature,
not a capability ceiling; (c) the reference gate holds at 121/121 at 7B
too. Stock JS *dropping* 89→68 is the 7B writing more elaborate (and more
often truncated/wrong) solutions to `-v2`-phrased tasks — raw outputs are
committed for anyone who wants to re-grade.
