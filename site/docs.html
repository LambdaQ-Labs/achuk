Claw documentation

Everything in one download: the compiler, the tooling, an AI model fine-tuned to write Claw, and the guardrails that verify every line it produces.

Install

curl -fsSL https://clawlang.dev/install.sh | sh

One command, no sudo, no system dependencies. It installs to ~/.claw and puts claw on your PATH. The bundle includes:

  • claw — the CLI (build, run, packages, AI)
  • clawc — the compiler (100–200 ms typechecks)
  • claw-mcp — the MCP server for AI assistants
  • claw-lsp — editor language server
  • claw-infer + model/claw-0.5b-q8.gguf — the bundled AI model and its inference engine

Update any time with claw upgrade. Windows: download the zip from GitHub Releases and add bin\ to PATH.

Linux note: claw run (compile-and-execute) needs a system linker — apt install binutils covers it. claw check and everything else need nothing.

Quick start

$ claw new hello --platform cli
$ cd hello
$ claw run main.claw
Hello, world!

$ claw check main.claw     # typecheck only — ~100 ms
$ claw fmt main.claw
$ claw test

Platforms decide what your program can touch: --platform cli for command-line apps, --platform http for a web service. The language itself has no I/O — that's what makes effects checkable.

The language in 60 seconds

# Functions: a signature, then a definition.
fizz : U64 -> Str
fizz = |n| {
    if n % 15 == 0 "FizzBuzz"
    else if n % 3 == 0 "Fizz"
    else if n % 5 == 0 "Buzz"
    else n.to_str()
}

# Tags and pattern matching — no null, no exceptions.
classify = |n| match n {
    0 => "zero"
    1 => "one"
    _ => "many"
}

# Effects are visible (the !) and checked. A pure function
# cannot quietly start reading files.
main! = |_args| {
    echo!(fizz(15))
    Ok({})
}

Functional, effect-typed, expression-oriented. Full walkthrough: the tour (15 minutes, runnable examples).

claw ai — generate code that's checked before you see it

$ claw ai gen "define atleast10 : Nat -> Nat that computes Nat.max of x and 10"
── generated ──
atleast10 : Nat -> Nat
atleast10 = |p0| Nat.max(p0, 10)
── verified ── real compiler: OK

What happened in that one command:

  1. The prompt was built from your project's real symbols (the code database) — the model sees what exists, not what it imagines.
  2. Generation ran grammar-constrained: names outside your project are impossible to emit at the token level.
  3. The result went through the actual compiler before display.

And when the model gets it wrong, you see this instead of a bug:

── REJECTED ── real compiler found 1 error(s)

Claw never hands you unverified generated code. Management commands:

claw ai status    # where the model and server are
claw ai serve     # start the model server (gen does this automatically)
claw ai stop

The model is a 0.5B fine-tune that runs on any laptop, CPU-only — no API key, no cloud, nothing leaves your machine.

The four gates

gatewhat it stops
1 · Code databaseguessing — the AI queries what exists (claw db candidates "Nat, Nat -> Nat") instead of inventing an API
2 · Decode grammarhallucination — out-of-scope names and undeclared effects are ungeneratable, not just discouraged
3 · Real typecheckstructural errors — wrong arity, undefined helpers, ignored parameters
4 · Executable contractswrong answers — code that typechecks but computes the wrong thing

No single gate is trusted. Measured effect: invented API calls per benchmark run went from 38 to 0, and the bundled model passes 121/121 tasks hallucination-free.

Claude, Cursor & MCP

Any MCP-capable assistant can drive Claw natively. Add to your client's MCP config:

{
  "mcpServers": {
    "claw": {
      "command": "~/.claw/bin/claw-mcp",
      "args": ["--db", "claw.cdb"]
    }
  }
}

Five tools become available: claw_symbols (what exists), claw_candidates (type-directed search), claw_mask (the decode grammar), claw_render (definition → source), and claw_check (real-compiler verdict on generated code). Per-client setup for Claude Code, Claude Desktop, Cursor, Windsurf, VS Code, Zed, Gemini CLI, and Codex CLI: docs/mcp.md.

Packages

$ claw add somelib          # fetch from registry.clawlang.dev
$ claw publish               # publish your own

Claw's registry has one rule no other registry has: every package must be AI-legible. Publishing exports the package's definitions — names, types, effect rows, docs — and the registry rejects packages without them. When you claw add a package, those definitions flow into your project's code database:

  • your editor's completions know the package immediately,
  • your AI assistant (MCP) can query it,
  • claw ai gen can call it — grammar-constrained, like everything else.

Install a package, and every tool — human-facing or AI-facing — understands it. No docs to scrape, no types to guess. Packages are content-addressed (identity = hash), which also makes typosquatting economically pointless.

Editor support

  • VS Code / Cursor / Windsurf: the Claw extension — syntax highlighting, snippets. Install the .vsix until it's on the marketplace.
  • Any LSP editor: point it at claw-lsp --db claw.cdb for completions and hover types.
  • AI assistants: the MCP server above.

The code database

Claw code lives in a queryable, content-addressed database — the thing that makes the AI honest and refactors instant.

$ claw index .                                # ingest your project
$ claw db symbols                              # everything that exists
$ claw db candidates "Str -> Str"              # type-directed search
$ claw db callers Nat.add                      # impact analysis
$ claw db eval myFunc 3 4                      # run a definition
$ claw db eval --real myFunc 3 4               # …through the actual compiler

For AI-written code, the verification commands:

$ claw defs-check out.json task.json           # real-compiler verdict
$ claw defs-grade out.json task.json           # + executed contracts, as JSON
$ claw task-grammar task.json                  # the decode grammar for a scope

Telemetry & privacy

  • Default: anonymous metrics only — command kinds, verdict flags, error counts. Never your code, paths, or prompts.
  • It announces itself on first use and one command turns it off: claw telemetry off (or CLAW_TELEMETRY=off). Off means zero writes.
  • Sharing code samples for training is strictly opt-in (claw telemetry full).
  • The local log is plain JSONL you can read (~/.claw/telemetry/), capped at 4 MiB.

Full policy: docs/telemetry.md.
